================================================================================
ThrowTrajectoryCorrector（投擲軌道補正）まとめ
================================================================================
対象スクリプト: Assets/Resources/Scripts/ThrowTrajectoryCorrector.cs
主な使用プレハブ: Assets/Resources/prefabs/NO BOM.prefab

--------------------------------------------------------------------------------
1. 目的
--------------------------------------------------------------------------------
キャラクターが投げたオブジェクトについて、
「投擲方向がターゲット方向に近いときだけ」軌道を補正し、命中しやすくする。

・移動速度は一定（重力なし）
・軌道は Y 軸固定（XZ 平面のみ移動。補正あり・なし共通）
・軌道は投擲時に一度だけ決定（飛行中に再計算しない）
・ターゲットが後から動いても、ゴール地点は更新しない

--------------------------------------------------------------------------------
2. 基本仕様（当初の要件）
--------------------------------------------------------------------------------
【投擲方向の定義】
  P0 … 投擲物が手（尻尾）から離れた瞬間の位置
  P1 … その次の FixedUpdate 時点の位置
  投擲方向 = normalize(P1 - P0)

【ターゲット方向】
  投擲時にターゲットが設定されている場合
  T … 投擲確定時点のターゲット位置（固定ゴール）
  ターゲット方向 = normalize(T - P0)

【補正判定】
  投擲方向とターゲット方向のなす角度を計算
  ・30度以上 → 補正しない（直進）
  ・30度未満 → 補正する（曲線で T へ）

【補正動作】
  ・ゴール T は投擲時に固定
  ・最初は投げた方向へ進み、徐々にカーブして T へ
  ・急な方向転換ではなくベジェ曲線による滑らかな弧
  ・軌道は投擲開始時にサンプリングして弧長テーブル化

--------------------------------------------------------------------------------
3. 実装の流れ
--------------------------------------------------------------------------------
1) 親 Transform が外れる（PlayerCatchEnemy で投げ離し）と飛行準備開始
2) P0 を記録し、ターゲット解決・ゴール位置をロック
3) 1 FixedUpdate 後に P1 を計測し投擲方向・速度を決定
4) 角度判定 → 直線 or ベジェ曲線のパスを構築
5) Rigidbody を Kinematic にし、一定速度でパス上を移動
6) 再キャッチ（親が付く）で飛行リセット・物理設定を復元

手動開始: NotifyThrown() も利用可能

--------------------------------------------------------------------------------
4. ターゲットの設定（プレハブで BOSS を入れられない理由）
--------------------------------------------------------------------------------
Project 上のプレハブ（NO BOM）から、Hierarchy の BOSS を
Throw Target にドラッグしても参照は保存できない（Unity の制約）。

【推奨設定（NO BOM 既定）】
  Find Target If Unset … ON
  Find By Tag … ON
  Target Tag … Boss
  → 投擲時に GameObject.FindGameObjectWithTag("Boss") で検索

  シーン内の BOSS オブジェクトに Tag: Boss が付いていること。

  名前検索を使う場合:
  Find By Tag … OFF
  Target Object Name … BOSS

  コードから指定:
  corrector.SetThrowTarget(bossGameObject);

--------------------------------------------------------------------------------
5. ゴール位置・飛行高さ（Y 軸固定）
--------------------------------------------------------------------------------
  飛行中の Y は Goal Height で固定（既定 1）。
  投擲方向・速度・軌道はすべて XZ 平面で計算する（上下には動かない）。

  ゴール位置:
    X, Z … ターゲットの Renderer / Collider 全体の中心（Bounds Center）
    Y    … Goal Height

  Use Target Bounds Center … ON 推奨

--------------------------------------------------------------------------------
6. Inspector 項目一覧
--------------------------------------------------------------------------------
【ターゲット】
  Throw Target          … 直接指定（任意・プレハブでは通常空）
  Find Target If Unset  … 未設定時にシーン検索
  Target Object Name    … 名前検索（Find 使用）
  Find By Tag           … タグ検索を優先
  Target Tag            … 既定 "Boss"

【ゴール位置】
  Use Target Bounds Center … ボス中心 XZ
  Goal Height              … ゴールの Y（既定 1）

【速度】
  Use Throw Measured Speed … ON: 投擲直後の実測速度 / OFF: 下の固定値
  Flight Speed             … 固定速度（Use Throw OFF 時）
  Speed Multiplier         … 最終速度への倍率（例 1.5 で1.5倍）

【補正判定】
  Max Correction Angle  … 補正する角度上限（既定 30）

【曲線軌道】
  Curve Tension         … カーブの強さ
  Path Sample Count     … 軌道サンプル数
  Max Flight Distance   … 非補正時の直進距離

【物理】
  Disable Gravity On Flight … 飛行中重力オフ
  Align Rotation To Velocity  … 進行方向へ向きを合わせる

【デバッグ】
  Draw Path Gizmo       … Scene 上に軌道を表示

--------------------------------------------------------------------------------
7. 速度の決め方（例）
--------------------------------------------------------------------------------
  投擲の勢いをそのまま使う:
    Use Throw Measured Speed = ON
    Speed Multiplier = 1

  常に一定速度で飛ばす:
    Use Throw Measured Speed = OFF
    Flight Speed = 15（任意）
    Speed Multiplier = 1

  投擲速度を少し強くする:
    Use Throw Measured Speed = ON
    Speed Multiplier = 1.2
